# 스테이지 직접 이동 설계

## 목표

상단 진행 경로의 각 스테이지 노드를 클릭하면 현재 진행도와 관계없이 해당 스테이지를 즉시 열어 플레이할 수 있게 한다.

## 선택한 접근

스테이지 노드 클릭에만 적용되는 직접 이동 경로를 추가한다. 기존 `goStep`의 순차 진행 규칙을 재사용하되, 노드 클릭임을 나타내는 옵션으로 현재 스테이지의 완료 조건 검사만 우회한다.

단순히 미래 노드 제한만 제거하는 방식은 Primer 또는 Background & Culture에서 `canLeaveForward()`에 다시 차단된다. 반대로 이전 활동을 강제 완료하는 방식은 사용자의 실제 완료 기록을 왜곡한다. 직접 이동 전용 옵션은 두 문제를 피하면서 기존 이동·렌더링 로직을 그대로 활용한다.

## 동작 규칙

1. `Ella Gator`, `Prologue`, `Primer`, `Chapter 1`, 중간 페이지, `Background & Culture`, `Review`를 포함한 현재 챕터의 모든 노드를 클릭할 수 있다.
2. 노드 클릭은 `maxStep`, `pollDone`, `activityDone` 상태와 관계없이 대상 단계로 이동한다.
3. 직접 이동만으로 `pollDone`, `activityDone`, `storyboardDone` 같은 완료 상태를 변경하지 않는다.
4. 일반 이전·다음 화살표는 기존 완료 조건과 순차 진행 규칙을 유지한다.
5. 기존에 명시적으로 잠긴 미래 챕터 노드가 추가될 경우 `locked` 상태는 계속 존중한다.
6. 대상 단계가 활동이면 기존 열기 함수로 즉시 플레이 화면을 연다.

## 구현 구조

- `nodeAction(index)`는 유효한 노드와 명시적 `locked` 상태를 확인한 뒤 직접 이동을 요청한다.
- `goStep(index, options)`는 `options.direct === true`일 때만 전방 완료 조건을 우회한다.
- 직접 이동에서는 이전 활동을 완료 처리하는 부수 효과를 실행하지 않는다.
- 단계 전환 이후의 활동 정리, 페이지 렌더링, 진행 경로 갱신, 화살표 갱신은 기존 `goStep` 흐름을 사용한다.

## 접근성

- 노드는 기존 네이티브 `<button>`을 유지해 키보드 Enter/Space 조작을 지원한다.
- 각 버튼에 챕터명 또는 페이지 라벨 기반의 `aria-label`을 제공한다.
- 미래 노드의 흐린 시각 표현은 진행 상태 표시로만 사용하며 클릭을 막는 의미로 사용하지 않는다.

## 오류 및 경계 조건

- 숫자가 아니거나 범위를 벗어난 인덱스는 무시한다.
- 명시적 `locked` 노드는 이동하지 않는다.
- 현재 단계 재클릭도 안전하게 동일 단계를 다시 연다.
- 직접 이동 후에도 미완료 활동의 완료 플래그는 유지되지 않아야 한다.

## 테스트 전략

- 미래 노드 클릭이 `direct` 옵션으로 `goStep`을 호출하는지 검증한다.
- 직접 이동 시 `canLeaveForward()` 차단과 이전 활동 자동 완료 처리가 적용되지 않는지 검증한다.
- 일반 이동에서는 기존 완료 조건과 자동 완료 흐름이 유지되는지 검증한다.
- 모든 노드 버튼에 유효한 접근성 이름이 생성되는지 검증한다.
- 전체 기존 테스트를 실행해 Storyboard와 Background & Culture 완료 흐름의 회귀가 없는지 확인한다.

## 범위 내 개선 검토

1. 클릭 가능 상태를 `aria-label`로 명확히 한다.
2. 직접 이동과 순차 이동을 옵션으로 구분해 완료 상태 오염을 막는다.
3. 문자열 존재 검사뿐 아니라 이동 분기를 실행하는 테스트를 추가한다.

세 항목 모두 이번 기능의 접근성 및 회귀 방지와 직접 관련되므로 구현에 반영한다.
