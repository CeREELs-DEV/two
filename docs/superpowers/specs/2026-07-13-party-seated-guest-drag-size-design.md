# 착석·드래그 게스트 크기 일치 설계

## 문제

드래그 중인 게스트 복제본은 `.stage` 밖에서 `scale(1.06)`으로 렌더링된다. 착석 게스트는 `.stage` 내부에서 화면 크기에 맞춰 함께 축소되므로, 같은 PNG라도 의자에 놓으면 더 작게 보인다.

## 선택한 방식

화면 맞춤 함수가 계산한 stage 배율을 `stageScale`이라고 할 때, 착석한 `.gav`에 `1.06 / stageScale` 확대율을 적용한다. stage 자체의 `stageScale`과 곱해진 최종 화면 배율은 드래그 복제본과 같은 `1.06`이 된다.

확대는 `.chair .gav`에만 적용한다. `.chair .guestwrap`의 `left:50%`와 `translateX(-50%)`는 유지해 의자 중앙 정렬을 보존하고, 말풍선과 드래그 좌표 계산에는 영향을 주지 않는다.

## 변경 범위

- Perfect Party 내장 문서의 stage 맞춤 함수가 CSS 변수 `--seated-guest-zoom`을 갱신
- `.chair .gav`가 해당 CSS 변수를 확대율로 사용
- 변환 스크립트가 기존 내장 문서를 새 CSS·JavaScript 규칙으로 마이그레이션
- 확대율 계산과 실제 내장 문서를 검증하는 회귀 테스트

PNG 매핑, 이미지별 crop 위치, 감정 상태, 드래그 복제본 크기 및 SVG 오류 fallback은 변경하지 않는다.

## 검증 기준

- `stageScale × seatedGuestZoom`은 항상 `1.06`이다.
- 착석 게스트의 중앙 정렬 규칙은 그대로 유지된다.
- 대기 중인 게스트에는 역보정 확대가 적용되지 않는다.
- 창 크기 변경과 최초 로드 시 확대율이 다시 계산된다.
- 기존 이미지 매핑과 변환 재실행 안정성 테스트가 모두 통과한다.
