# 음식 이미지 및 스토리 레이아웃 보정 설계

## 목표

사용자가 제공한 새 Peking duck·dimsum 그림을 Perfect Party에 교체하고, P13의 `alone…”`/`No one` 시각적 겹침과 `The egg is`의 불필요한 굵은 표시를 제거한다.

## 선택한 접근

새 음식 그림은 투명 픽셀 경계로 크롭한 후 비율을 유지해 최대 `210×210` 안에 맞추고, `240×240` 투명 캔버스 중앙에 배치한다. 결과 PNG는 외부 자산으로 저장하고 Perfect Party의 두 `FOODIMG` 항목만 해당 경로로 바꾼다.

P13은 글자 크기나 행간을 변경하지 않고 `No one notices...`가 포함된 블록의 세로 시작점만 조정한다. `The egg is`는 문구와 위치를 유지하면서 `.lead`의 글꼴 굵기만 일반 굵기로 바꾼다.

## 검토한 대안

### 음식 이미지

1. 원본 `2500×1252` 이미지를 그대로 사용하고 CSS에서 확대·좌표 보정
   - 음식 카트, 테이블, 선호도 말풍선마다 서로 다른 크기 보정이 필요하고 원본의 큰 투명 여백 때문에 유지보수가 어렵다.
2. 정규화한 결과를 base64로 다시 내장
   - 기존 `FOODIMG` 형식과 같지만 `index.html`이 커지고 이미지 변경 이력을 확인하기 어렵다.
3. 정규화한 외부 PNG 경로 사용
   - 파일 단위 검증과 재교체가 쉽고 내장 HTML 크기를 늘리지 않는다. 이 방식을 선택한다.

### P13 겹침

1. 글자 크기 또는 행간 축소
   - 겹침은 줄지만 페이지 전체의 가독성과 다른 본문 규칙이 달라진다.
2. `No one...` 블록만 아래로 이동
   - 원인을 직접 해결하면서 다른 문단에 영향을 주지 않는다. 이 방식을 선택한다.
3. `“Oh well...”`과 `No one...`을 하나의 흐름 레이아웃으로 통합
   - 장기적으로 유연하지만 현재 절대 좌표 기반 페이지 구조를 크게 변경한다.

### `The egg is` 굵기

1. `.lead`의 굵기를 일반 굵기로 변경
   - P3의 `She’s`는 이미 lead에서 제거되어 현재 실제로 남는 lead는 `The egg is` 하나다. 이 방식을 선택한다.
2. 문장별 일반 굵기 플래그 추가
   - 단일 문구를 위해 데이터·렌더러·CSS가 함께 복잡해진다.
3. `The egg is`를 첫 강조 조각에 포함
   - 문구 전체가 하이라이트되어 요청 범위를 넘는다.

## 이미지 자산 규칙

- 입력 Peking duck: `/Users/suhyeon/Downloads/Foot_CH_Duck 2.png`
- 입력 dimsum: `/Users/suhyeon/Downloads/Foot_CH_Dumpling.png`
- 출력 Peking duck: `images/Peking_Duck.png`
- 출력 dimsum: `images/DimSum.png`
- 출력 크기: 정확히 `240×240` RGBA PNG
- 그림 크기: 투명 경계를 기준으로 가로·세로 모두 최대 `210px`
- 배치: 투명 캔버스의 가로·세로 중앙
- 비율: 원본 종횡비 유지
- 원본 그림의 색상과 형태는 변경하지 않는다.

Perfect Party의 `FOODIMG.f_fish`는 `images/Peking_Duck.png`, `FOODIMG.f_dumpling`은 `images/DimSum.png`를 참조한다. 나머지 음식 이미지 값은 변경하지 않는다.

## P13 레이아웃 규칙

P13의 페이지 이미지 비율은 `763×1252`이며 `“Oh well...”` 한 줄의 렌더링 높이는 페이지 높이의 약 `0.04623`이다. 현재 다음 블록까지 남는 간격은 약 `0.00687`로 너무 작다.

`No one notices Ella sitting off`가 포함된 블록의 `y`를 `0.6321`에서 `0.6585`로 변경한다. 변경 후 두 블록의 간격은 약 `0.03327`이고, 네 줄 본문의 하단은 약 `0.84343`이므로 페이지 내부에 유지된다.

다른 P13 블록의 위치, 글자 크기, 행간, 들여쓰기는 변경하지 않는다.

## `The egg is` 표시 규칙

- 문구 `The egg is`는 그대로 표시한다.
- 문장 분해 후 별도 lead로 유지한다.
- `.lead`의 `font-weight`를 `700`에서 `400`으로 변경한다.
- 강조 배경은 뒤따르는 단순화 조각에만 적용한다.

## 구현 경계

- 음식 PNG 2개를 추가한다.
- Perfect Party 내장 문서의 `FOODIMG` 두 값만 갱신한다.
- P13의 `No one...` 블록 좌표와 `.lead` 굵기만 변경한다.
- 음식 이름, 점수 계산, 드래그 앤 드롭, 기존 게스트 이미지, 다른 페이지 레이아웃은 변경하지 않는다.

## 테스트 및 검증

1. 새 음식 PNG가 존재하며 PNG 헤더 크기가 각각 `240×240`, 색상 형식이 RGBA인지 검증한다.
2. 이미지 정규화 후 투명 경계가 최대 `210×210`이고 캔버스 중앙에 있는지 이미지 도구로 검증한다.
3. 내장 Perfect Party의 두 `FOODIMG` 경로가 새 파일을 가리키고 나머지 음식 매핑이 유지되는지 검증한다.
4. P13의 대상 블록 `y`가 `0.6585`이고 계산된 문단 간격이 `0.03` 이상인지 검증한다.
5. `.lead`의 `font-weight`가 `400`이고 `The egg is` 문구가 유지되는지 검증한다.
6. 전체 기존 테스트와 인라인 JavaScript 구문 검사를 실행한다.

## 개선 사항 검토 및 반영

1. 이미지 데이터를 거대한 HTML 문자열에 다시 넣지 않고 파일로 분리해 변경 검토와 캐시 활용을 개선한다.
2. 단순 문자열 검사뿐 아니라 이미지 크기와 P13 실제 간격 계산을 테스트해 재발을 방지한다.
3. 두 음식 이외의 `FOODIMG` 값이 유지되는지 확인해 범위 밖 자산 변경을 막는다.

세 항목은 이번 변경의 안전성 및 유지보수성과 직접 관련되므로 모두 반영한다.
