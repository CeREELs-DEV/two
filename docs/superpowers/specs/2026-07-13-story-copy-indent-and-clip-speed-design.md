# 스토리 문구·들여쓰기·클립 속도 설계

## 목표

P3·P4·P10·P13의 지정 문구와 강조·들여쓰기를 정확히 보정하고, Prologue 및 P6-7 Storyboard의 클립 속도 버튼이 실제 영상 재생 속도를 변경하게 한다.

## 선택한 접근

문장 내용은 원본 페이지 배열의 위치가 아니라 정확한 문구를 기준으로 보정한다. Storyboard 재생 속도는 공통 영상 플레이어에 `applyPlayerSpeed()`를 두고 속도 버튼 클릭, 영상 프레임 로드, 재생 재개 경로에서 같은 값을 적용한다.

이 방식은 전역 CSS를 변경해 다른 문장의 굵기까지 바꾸는 문제와, 페이지·블록 인덱스에 결합되어 데이터 순서가 바뀔 때 잘못된 문장을 수정하는 문제를 피한다.

## 문구 및 강조 규칙

1. P3의 `She’s a little awkward...` 인터랙션은 별도 굵은 선행어 `She’s`를 제거한다.
2. 같은 문장의 첫 강조 조각을 `She’s awkward`로 변경해 두 단어를 하나의 하이라이트로 표시한다.
3. P10의 단순화 문구 `Ella walks up to Uncle Jack`을 `Ella walks to Uncle Jack`으로 변경한다.
4. 원문 문장과 나머지 단순화 조각은 변경하지 않는다.

## 들여쓰기 규칙

기존 본문에 사용 중인 표준 들여쓰기 비율 `0.06015`를 다음 문단의 첫 줄에 적용한다.

- P4 `She can’t contain her excitement as she sings:`
- P13 `But Uncle Jack doesn’t recognize poor Ella.`
- P13 `“Who are you? And what are you?” ...`
- P13 `“Has his memory gotten that bad, ...` 
- P13 `“Oh well, I guess I’ll just eat alone…”`
- P13 `No one notices Ella sitting off`

문단의 정확한 전체 텍스트를 키로 사용해 다른 페이지의 문장에는 영향을 주지 않는다.

## Storyboard 규칙

1. 단독 Storyboard 범위 라벨은 `p. 2-3`, `p. 6-7`로 표시한다. 현재 단독 Storyboard가 이 두 범위뿐이므로 단독 탭 형식을 `p.`로 통일한다.
2. 속도 선택값 `0.5×`, `1×`, `1.5×`, `2×`를 실제 `<video>` 요소의 `playbackRate`에 반영한다.
3. 재생 중 속도 버튼을 누르면 현재 클립에 즉시 반영한다.
4. 다음 클립으로 넘어가거나 일시 정지 후 재생해도 선택한 속도를 유지한다.
5. 정지 이미지 프레임의 체류시간은 기존처럼 선택 배율에 반비례한다.

## 테스트 전략

- `INTERACT_B`와 페이지 보정 함수를 VM에서 실행해 P3·P10 문구와 6개 문단 들여쓰기를 검증한다.
- 단독 Storyboard 탭 생성 결과가 `p. 2-3`, `p. 6-7`인지 검증한다.
- 가짜 영상 요소에 `applyPlayerSpeed()`를 실행해 네 속도 값이 정확히 반영되는지 검증한다.
- 속도 버튼·영상 로드·재생 재개 경로가 공통 속도 함수를 호출하는지 검증한다.
- 전체 Node 테스트와 HTML 내장 JavaScript 구문 검사를 실행한다.

## 범위 내 개선 검토

1. 문구 기반 보정 함수를 사용해 페이지 인덱스 변경에 대한 내성을 높인다.
2. 속도 적용을 단일 함수로 모아 새 재생 경로가 추가될 때 누락 가능성을 줄인다.
3. DOM 문자열 존재 검사뿐 아니라 데이터와 속도 함수를 실제 실행하는 회귀 테스트를 추가한다.

세 항목 모두 이번 수정의 정확성과 재발 방지에 직접 관련되므로 구현에 반영한다.
